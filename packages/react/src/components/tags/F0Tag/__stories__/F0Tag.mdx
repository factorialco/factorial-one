# Tag - Router Component

The `F0Tag` component is a **router component** that renders different tag types
based on a `type` property.

## When to Use F0Tag

The F0Tag router component should only be used in these scenarios:

- When building dynamic UI where the tag type is determined at runtime
- When integrating with APIs that return tag configuration objects
- When building higher-order components that need to render various tag types

## How F0Tag Works

F0Tag accepts a `tag` prop with a `TagVariant` type that includes:

- A `type` property that determines which specific tag component to render
- All the props required by the specific tag component
- An optional `description` property for tooltip content

## Best Practices

1. **Use specific tag components** instead of F0Tag for better developer
   experience
2. **Only use F0Tag** when you need dynamic tag rendering based on runtime data
3. **Prefer composition** over the router pattern when building static UIs
4. **Always provide proper TypeScript types** when using the TagVariant union
   type

## Type Safety

The F0Tag component uses a discriminated union type (`TagVariant`) that ensures
type safety across all tag variants. Each tag type has its own specific props
that are enforced at compile time.

```typescript
type TagVariant =
  | BaseTag<{ type: "dot" } & TagDotProps>
  | BaseTag<{ type: "person" } & PersonTagProps>
  | BaseTag<{ type: "team" } & TeamTagProps>
  | BaseTag<{ type: "company" } & CompanyTagProps>
  | BaseTag<{ type: "alert" } & AlertTagProps>
  | BaseTag<{ type: "status" } & TagStatusProps>
  | BaseTag<{ type: "balance" } & BalanceTagProps>
  | BaseTag<{ type: "raw" } & TagRawProps>
```
